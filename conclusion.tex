\section{Conclusion and Future Work}
\label{sec:conclusion}

% We proposed a proof logging approach based on symbolic execution
% which generates safety proofs for LLVM IR programs with memory operations.
% Implemented using Rocq/KLEE
% Evaluated on APIs from real-world benchmarks
% SE is an effective mechanism for generating proofs

\paragraph{Future Work}
% Modeling the environment
Currently, our formalization does not support external calls.
Extending our approach with environment modeling would further broaden its applicability.
% Scaling proof compilation
Larger programs generally yield larger proofs, which take more time to check.
Accelerating proof checking would significantly enhance the scalability of our approach.
% Implementing verified SE engines
While our formalization is currently used for proof logging,
it can be leveraged to implement a verified SE engine via code extraction from proof assistants.
% Concrete semantics
The concrete semantics can be used as a foundation for generating proofs using other techniques,
such as deductive verification and abstract interpretation.
% Symbolic semantics
The symbolic semantics can be used to formalize advanced extensions of symbolic execution,
including state merging and alternative memory models.
